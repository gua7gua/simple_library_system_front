## 二、通用响应格式

### 1. ApiResponse（操作结果响应）

**使用场景**：登录、登出、获取用户信息

```json
{
    "status": 200,
    "message": "操作成功",
    "data": {},
    "timestamp": 1234567890
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| status | int | 状态码（200成功，420失败） |
| message | String | 消息 |
| data | Object | 数据 |
| timestamp | long | 时间戳 |

### 2. PageResponse（分页查询响应 / 单值响应）

**使用场景A**：分页查询接口

```json
{"code": 0, "message": "success", "count": 100, "data": []}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码（0成功，1失败） |
| message | String | 消息 |
| count | long | 总记录数 |
| data | Array | 数据列表 |

**使用场景B**：文件上传等单值返回接口

```json
{"code": 0, "message": "success", "count": 0, "singleData": "/upload/xxx.jpg"}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 状态码（0成功，1失败） |
| message | String | 消息 |
| count | long | 固定为0 |
| singleData | String | 单个值（如文件路径），仅成功时返回 |

### 3. Integer（增删改操作响应）

**使用场景**：注册、添加、删除、更新操作

```json
1
```

成功返回 `1`，失败返回 `0`

## 三、用户管理模块 `/user`

### 1. 用户登录

| 项目 | 内容 |
|------|------|
| 接口地址 | POST /user/login |
| 是否需要认证 | 否 |
| Content-Type | application/json |

**请求参数**：

```json
{"username": "admin", "userpassword": "123456", "isadmin": 1}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | String | 是 | 用户名 |
| userpassword | String | 是 | 密码 |
| isadmin | Byte | 是 | 角色标识（0-读者，1-管理员） |

**成功响应**：

```json
{"status": 200, "message": "登录成功", "data": {"token": "a1b2c3d4e5f6..."}, "timestamp": 1711180800}
```

**失败响应**：

```json
{"status": 420, "message": "用户名或密码错误，或角色不匹配", "data": null, "timestamp": 1711180800}
```

> - 使用事务保证数据一致性

## 七、文件上传模块 `/update`

### 1. 上传图片

| 项目 | 内容 |
|------|------|
| 接口地址 | POST /update/updateImg |
| 是否需要认证 | 是 |
| Content-Type | multipart/form-data |

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| file | File | 是 | 图片文件 |

**文件限制**：

| 限制项 | 值 |
|--------|-----|
| 允许格式 | JPG、PNG |
| 文件大小 | 最大 10MB |

**成功响应**：

```json
{"code": 0, "message": "success", "count": 0, "singleData": "/upload/20260410_a1b2c3d4.jpg"}
```

**失败响应**：

```json
{"code": 1, "message": "文件大小超过10MB限制"}
```

> 说明：
> - 文件名会自动添加时间戳防止重复
> - 返回的URL可直接用于图书的 `bookimg` 字段
> - 文件存储在配置的 `uploads` 目录下
> - 成功时 data 为单个字符串（使用 singleData 字段）

## 十一、错误码说明

| 错误码 | 说明 | 常见原因 |
|--------|------|----------|
| status: 200 | 操作成功 | - |
| status: 420 | 操作失败 | 业务逻辑验证失败 |
| code: 0 | 查询成功 | 分页接口成功 |
| code: 1 | 操作失败 | 分页接口操作失败 |
| 返回 0 | 操作失败 | 增删改接口失败 |
| 返回 1 | 操作成功 | 增删改接口成功 |

### 常见错误信息

| 错误信息 | 说明 |
|----------|------|
| 用户名或密码错误，或角色不匹配 | 登录凭证不正确 |
| Token无效或已过期 | Token验证失败 |
| 权限不足，需要管理员权限 | 普通用户访问管理员接口 |
| 图书已借出 | 尝试借阅已被借出的图书 |

| 旧密码错误 | 修改密码时旧密码验证失败 |
| 文件大小超过10MB限制 | 上传文件过大 |
| 仅支持 JPG、PNG 格式的图片 | 文件类型不支持 |
| 捐赠申请不存在 | 操作的捐赠申请ID不存在 |

## 十三、业务规则汇总

### 用户管理

- 用户名必须唯一
- 密码长度建议6-20位
- 注册时默认创建为普通读者角色（isadmin=0）

### 图书管理

- 图书类型ID必须存在于book_type表中
- 已借出的图书不允许删除
- 添加图书时isborrowed默认值为0（可借）

### 借阅管理

- 图书必须处于"可借"状态才能借阅
- 已归还的借阅记录不允许重复归还
- 借书和还书操作使用事务保证数据一致性
- 并发借阅同一本书时，只有一个请求会成功

### 文件上传

- 仅支持JPG、PNG格式
- 文件大小最大10MB
- 文件名自动添加时间戳防止重复

### 图书捐赠

- 捐赠人姓名、手机号、图书ISBN为必填项
- 捐赠申请提交后状态默认为"待处理"（status=0）
- 状态更新为"待接收"时自动记录联系时间
- 状态更新为"已入库"时自动记录接收时间
- 管理员可以添加工作人员备注
- 捐赠人可以根据手机号查询自己的捐赠记录

## 十四、接口调用示例

### 完整借书流程

```bash
# 1. 用户登录
curl -X POST "http://localhost:8092/BookManager/user/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"reader01","userpassword":"123456","isadmin":0}'
# 返回: {"status":200,"data":{"token":"abc123..."}}

# 2. 查询可借图书
curl "http://localhost:8092/BookManager/bookInfo/queryBookInfosByPage?page=1&limit=10&bookname=Java&token=abc123"

# 3. 借书
curl "http://localhost:8092/BookManager/borrow/reader/borrowBook?userid=2&bookid=5&token=abc123"
# 返回: 1 (成功)

# 4. 查询我的借阅记录
curl "http://localhost:8092/BookManager/borrow/queryBorrowsByPage?page=1&limit=10&userid=2&token=abc123"

# 5. 还书
curl "http://localhost:8092/BookManager/borrow/reader/returnBook?borrowid=1&bookid=5&token=abc123"
# 返回: 1 (成功)

# 6. 登出
curl "http://localhost:8092/BookManager/user/logout?token=abc123"
```
